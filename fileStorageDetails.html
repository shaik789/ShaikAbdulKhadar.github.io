PORTFOLIO FILE STORAGE - Details

Description

We are very well aware of encryption techniques and strength of private key encryption. Today we are using the same private key encryption technique to secure our files. We are also noticing the advancements in programming using python. Here we are merging these two techniques, cryptography and python, to make a secure file storage location using hybride cryptography.

The purpose model is liable to meet the required security needs of data centre of cloud. Blowfish algorithm is used for the encryption of file slices which takes minimum time to complete and has maximum throughput for encryption and decryption from other symmentric algorithm. The idea of splitting and merging adds on to meet the principle of data security. The hybride approach when deployed in cloud environment makes the remote server more secure and thus, helps the cloud providers to fetch more trust of their users. For data security and privacy protection issues, the fundamental challenges of separation of sensitive data and access control is fulfilled. Cryptography technique translates original data into unreadable form. Cryptography technique is divided into symmetric key cryptography and public key cryptography. This technique uses keys for translate data into unreadable form. So only authorized person can access data from cloudserver. Cipher text data is visible for all people.

When you run the python program file, it stores your files in encrypted format in a new folder named uploads in the same folder as the python file itslef. It will also provide you with a file having key with extension ".pem" which can be used for decoding the encrypted file when ever needed. After encryption, any time you want your file back then simply use key file and you will be provided with the original file in a downloadable format.
